# Задание 1

## Настройка окружения проекта
- Открыть командную строку в папке проекта.
- Запустить в текущей папке [setup.bat](./setup.bat). Будут установлены виртуальное окружение и зависимости из [requirements.txt](requirements.txt). 
Можно пропустить данный пункт, если у вас настроена свое окружение, но зависимости брать указанные.
- Создать/отредактировать файл [.env](.env) с переменными окружения:
  - DEBUG - режим отладки - устанавливаем (on), пока ведется разработка приложения;
  - SECRET_KEY - используется Django при создании хэшей;
  - __DATABASE_HOST__ - сервер/адрес базы данных;
  - DATABASE_PORT - порт подключения к серверу;
  - __DATABASE_NAME__ - имя базы данных;
  - DATABASE_USER - логин пользователя для подключения;
  - DATABASE_PASSWORD - пароль пользователя для подключения.
  
  Обязательные для БД - DATABASE_HOST и DATABASE_NAME. При использовании localdb остальные переменные для подключения к бд можно не использовать
  (если подразумевается аутентификация windows).

## Создание пустой БД
В папке проекта запускаем команду:
```
python manage.py createdb
``` 
Создается новая бд с именем, указанным в __DATABASE_NAME__.
Если указанная БД уже существуют, то команда завершится неудачей (выбросится исключение).
Чтобы пересоздать БД (предварительно удалив старую), указываем ключ `-f` (или `--force`):
```
python manage.py createdb -f
```

## Миграции и запуск
В папке проекта запускаем команды:
- `python manage.py makemigrations` - создание миграций на основе созданных моделей;
- `python manage.py migrate` - проведение созданных миграций в БД;
- `python manage.py createsuperuser` - добавление суперпользователя.

После добавления суперпользователя сможем войти в админку, для этого:
- `python manage.py runserver` - запускаем сервер
- по адресу http://127.0.0.1:8000/admin/ заходим в админку.
Можем убедиться, что модель пользователя добавлена в админку.

# Задание 2

## Добавление зависимостей
В файле [requirements.txt](requirements.txt) добавили библиотеку `pillow` для работы с изображениями.
Для добавления ее в виртуальное окружение можно выполнить команду:
```
pip install pillow
```
Либо обновить пакеты согласно файлу зависимостей:
```
pip install -U -r requirements.txt
```

## Добавление миграций и запуск
На данном этапе обновлена модель пользователей и добавлены модели собак и пород.
В папке проекта запускаем команды:
- `python manage.py makemigrations` - создание миграций на основе созданных и обновленных моделей;
- `python manage.py migrate` - проведение созданных миграций в БД;
- `python manage.py runserver` - запускаем сервер.

Кроме стартовой страницы http://127.0.0.1:8000/ добавлены следующие маршруты:
- http://127.0.0.1:8000/breeds/ - список пород;
- http://127.0.0.1:8000/breeds/1/ - описание породы (вместо `1` может быть любое число);
- http://127.0.0.1:8000/dogs/ - список собак;
- http://127.0.0.1:8000/dogs/1/ - описание собаки (вместо `1` может быть любое число);
